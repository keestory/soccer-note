## 사용 가이드

### 테스트 우선순위
1. **Critical**: 인증, 데이터 무결성, 보안
2. **High**: 핵심 기능, 권한 관리
3. **Medium**: UI/UX, 반응형
4. **Low**: 성능 최적화, 부가 기능

### 효율적인 테스트 방법
1. **자동화**: 주요 시나리오는 E2E 테스트 작성 (Playwright)
2. **수동 테스트**: 복잡한 UI/UX 흐름
3. **크로스 브라우저**: BrowserStack 활용 (선택사항)

### 버그 리포팅
- **재현 단계** 명확히 기록
- **스크린샷** 첨부
- **기대 동작 vs 실제 동작** 설명

### MVP 테스트 최소 범위
- 인증 시스템 완벽히 테스트
- 핵심 기능 1개당 최소 5개 테스트 케이스
- 모바일 반응형 필수 확인
- 보안(RLS) 테스트 필수
