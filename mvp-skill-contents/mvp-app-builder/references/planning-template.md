## 작성 가이드

### 필수 요소
1. **명확한 가치 제안**: 왜 이 앱이 필요한가?
2. **구체적인 기능**: 추상적이지 않고 실제 구현 가능한 기능
3. **현실적인 일정**: 1개월 내 완성 가능한 범위

### MVP 범위 설정 팁
- **포함**: 핵심 가치를 전달하는 최소 기능
- **제외**: 부가 기능, 고급 기능, 최적화
- **원칙**: "없으면 앱이 의미 없어지는 기능"만 포함

### 좋은 사용자 스토리 예시
- ✅ "독서 애호가로서, 읽은 책을 쉽게 기록하기 위해, 책 제목 검색과 별점 입력을 하고 싶다"
- ❌ "사용자가 책을 관리할 수 있다" (너무 추상적)

### 기술 스택 선택 기준
1. **학습 곡선**: 이미 알거나 빠르게 배울 수 있는가?
2. **생산성**: 빠른 개발이 가능한가?
3. **생태계**: 필요한 라이브러리/도구가 있는가?
4. **배포 용이성**: 쉽게 배포할 수 있는가?

## 예시: 독서 기록 앱

```markdown
# BookTrack - MVP 기획안

## 1. 프로젝트 개요

### 주제
독서 습관을 기록하고 AI가 맞춤 추천을 제공하는 독서 관리 플랫폼

### 타겟 사용자
- **주요 사용자**: 월 1권 이상 책을 읽는 25-40대 직장인
- **사용 시나리오**: 책을 다 읽은 직후, 잠들기 전 독서 기록

### 핵심 가치 제안
1. 간편한 독서 기록 (30초 이내 완료)
2. AI 기반 개인화 도서 추천
3. 독서 패턴 분석 및 시각화

### 차별화 포인트
기존 독서 앱은 단순 기록에 그치지만, BookTrack은 AI가 사용자의 취향을 학습하여 능동적으로 다음 책을 추천

## 2. 핵심 기능 (MVP 범위)

### 필수 기능
1. **책 검색 및 등록**
   - 설명: 네이버/구글 Books API로 책 정보 가져오기
   - 사용자 스토리: "독서가로서, 방금 읽은 책을 빠르게 등록하기 위해, 책 제목만 입력하고 자동완성으로 선택하고 싶다"
   - 우선순위: High

2. **독서 기록 작성**
   - 설명: 별점, 한줄평, 독서 시작/완료일 기록
   - 사용자 스토리: "독서가로서, 나의 생각을 기록하기 위해, 간단한 별점과 한줄평을 남기고 싶다"
   - 우선순위: High

3. **AI 도서 추천**
   - 설명: 사용자의 독서 이력 기반 Claude API 추천
   - 사용자 스토리: "독서가로서, 다음에 읽을 책을 고르기 위해, 내 취향에 맞는 추천을 받고 싶다"
   - 우선순위: High

4. **독서 통계**
   - 설명: 월별 독서량, 장르 분포 차트
   - 사용자 스토리: "독서가로서, 나의 독서 습관을 돌아보기 위해, 통계를 보고 싶다"
   - 우선순위: Medium

### 향후 고려 기능 (MVP 이후)
- 소셜 기능 (친구 추가, 독서 모임)
- 독서 챌린지 (연간 목표 설정)
- 독서 노트 (메모, 인용구 저장)

## 3. 기술 스택

### 프론트엔드
- **프레임워크**: Next.js 14 (App Router)
- **스타일링**: Tailwind CSS
- **차트**: Recharts
- **UI**: shadcn/ui

### 백엔드
- **BaaS**: Supabase
  - PostgreSQL
  - Auth (Email + Google OAuth)
  - RLS 정책
  - Edge Functions (AI 추천 로직)

### API
- **도서 검색**: 네이버 Books API
- **AI 추천**: Claude API (Anthropic)
```
