# MVP App Builder - 1개월 완성 가이드

1개월 안에 MVP 앱을 만들기 위한 체계적인 7단계 워크플로우.

## 개발 스택

- **개발 도구**: Claude Code
- **백엔드**: Supabase (DB + Auth + Storage + API)
- **프론트엔드**: React/Next.js (또는 사용자 선호 프레임워크)
- **디자인**: 미니멀 디자인 (Tailwind CSS)

## 워크플로우 실행 방법

사용자로부터 다음 정보를 받은 후 7단계를 순차적으로 진행:
1. **주제** (예: "운동 기록 앱", "독서 관리 플랫폼")
2. **핵심 솔루션** (예: "AI 기반 운동 추천", "독서 습관 트래킹")

각 단계는 **완료 확인 후** 다음 단계로 진행. 사용자가 스킵을 요청하면 해당 단계 건너뛰기 가능.

## 7단계 워크플로우

### 1단계: 기획안 작성

**목표**: Claude Code가 이해할 수 있는 명확한 기획 문서 생성

**실행**:
- `references/planning-template.md` 참조하여 기획안 작성
- 결과물: `PROJECT_PLAN.md` 파일 생성 (프로젝트 루트)

**포함 내용**:
- 앱 개요 (주제, 타겟 사용자, 핵심 가치)
- 핵심 기능 3-5개 (MVP에 필수적인 것만)
- 기술 스택 명시
- 1개월 일정 (주차별 마일스톤)

**완료 조건**: `PROJECT_PLAN.md` 파일이 생성되고 사용자 승인

---

### 2단계: 시스템/운영 정책 설계

**목표**: 데이터베이스 스키마, API 설계, 인증/권한 정책 정의

**실행**:
- `references/system-design-template.md` 참조
- Supabase 기반 시스템 아키텍처 설계
- 결과물: `SYSTEM_DESIGN.md` 파일 생성

**포함 내용**:
- 데이터베이스 스키마 (테이블, 관계, 제약조건)
- API 엔드포인트 목록
- 인증/권한 정책 (Row Level Security)
- 파일 스토리지 정책 (필요시)

**완료 조건**: `SYSTEM_DESIGN.md` 파일 생성 및 사용자 승인

---

### 3단계: 디자인 작업

**목표**: 미니멀하고 실용적인 UI/UX 디자인

**실행**:
- `references/design-guidelines.md` 참조
- 간단한 와이어프레임과 디자인 시스템 정의
- 결과물: `DESIGN_SPEC.md` 파일 생성

**포함 내용**:
- 주요 화면 와이어프레임 (텍스트 기반 또는 ASCII)
- 컬러 팔레트 (3-5개 색상)
- 타이포그래피 (폰트 패밀리, 크기 시스템)
- 주요 컴포넌트 목록

**시간 절약 팁**: Tailwind CSS 기본 테마 활용

**완료 조건**: `DESIGN_SPEC.md` 파일 생성 및 사용자 승인

---

### 4단계: 서버 작업 및 API 개발

**목표**: Supabase 프로젝트 설정 및 API 구현

**실행**:
- `references/supabase-setup.md` 참조
- Supabase 프로젝트 생성 및 설정
- 결과물: `supabase/` 디렉토리 및 마이그레이션 파일

**작업 순서**:
1. Supabase 프로젝트 생성 (사용자가 직접 supabase.com에서 생성)
2. 데이터베이스 마이그레이션 파일 작성
3. Row Level Security (RLS) 정책 작성
4. Edge Functions 작성 (필요시)
5. Storage 버킷 설정 (필요시)

**디렉토리 구조**:
```
supabase/
├── migrations/
│   └── 001_initial_schema.sql
├── functions/
│   └── [function-name]/
└── seed.sql (선택사항)
```

**완료 조건**: 
- Supabase 로컬 개발 환경 또는 클라우드 프로젝트 설정 완료
- 마이그레이션 실행 성공
- API 테스트 성공

---

### 5단계: 클라이언트 작업 (프론트엔드 개발)

**목표**: 사용자 인터페이스 구현

**실행**:
- `references/frontend-patterns.md` 참조
- React/Next.js 앱 생성 및 개발
- 결과물: `src/` 또는 `app/` 디렉토리 및 컴포넌트

**작업 순서**:
1. 프로젝트 초기화 (Next.js 권장)
2. Supabase 클라이언트 설정
3. 인증 플로우 구현
4. 주요 기능 페이지 개발
5. Tailwind CSS로 스타일링

**디렉토리 구조** (Next.js App Router 예시):
```
app/
├── (auth)/
│   ├── login/
│   └── signup/
├── (dashboard)/
│   ├── layout.tsx
│   └── [feature]/
├── api/
└── layout.tsx
components/
└── ui/
lib/
└── supabase.ts
```

**완료 조건**:
- 주요 기능 페이지 동작 확인
- Supabase 연동 테스트 성공
- 로컬 환경에서 실행 가능

---

### 6단계: 셀프 테스트를 위한 QA 리스트 제작

**목표**: 체계적인 테스트로 버그 최소화

**실행**:
- `references/qa-checklist.md` 참조
- 프로젝트별 맞춤 체크리스트 생성
- 결과물: `QA_CHECKLIST.md` 파일

**포함 내용**:
- 기능 테스트 (각 핵심 기능별)
- 인증/권한 테스트
- UI/UX 테스트
- 반응형 테스트
- 엣지 케이스 테스트
- 성능 테스트 (기본적인 것만)

**완료 조건**: 
- `QA_CHECKLIST.md` 파일 생성
- 주요 테스트 항목 실행 및 통과

---

### 7단계: 앱스토어/웹 배포를 위한 준비

**목표**: 프로덕션 배포 준비 완료

**실행**:
- `references/deployment-guide.md` 참조
- 배포 플랫폼별 설정
- 결과물: `DEPLOYMENT.md` 파일 및 배포 완료

**웹 앱 배포** (Vercel 권장):
1. 환경 변수 설정
2. Vercel 프로젝트 연결
3. 프로덕션 배포
4. 커스텀 도메인 연결 (선택사항)

**모바일 앱 배포** (React Native 사용 시):
1. 앱 아이콘 및 스플래시 스크린
2. 앱스토어 메타데이터 준비
3. 빌드 및 테스트
4. 스토어 제출

**포함 내용**:
- 배포 체크리스트
- 환경 변수 목록
- 배포 명령어
- 롤백 절차

**완료 조건**:
- 프로덕션 환경 배포 성공
- 실제 URL 접근 가능
- 기본 모니터링 설정

---

## 진행 관리

각 단계마다:
1. 단계 시작 전 요구사항 확인
2. 참조 문서 확인 (`references/` 디렉토리)
3. 결과물 생성
4. 사용자 검토 및 승인
5. 다음 단계로 진행

**중단/재개**: 
- 진행 상황은 생성된 파일들로 추적
- 언제든 중단 후 다시 시작 가능
- "3단계부터 시작" 같은 부분 재개 지원

**유연성**:
- 사용자가 특정 단계 건너뛰기 요청 시 허용
- 단계 순서 변경 가능 (단, 의존성 고려)
- 각 단계 결과물 수정/재작성 가능

## 시작 명령어 예시

```
주제: 독서 기록 앱
솔루션: AI 기반 독서 추천과 진행률 트래킹

위 내용으로 MVP 앱 개발 시작해줘.
```

또는 단계별 시작:
```
3단계 디자인부터 시작할게. 기획안과 시스템 설계는 이미 있어.
```
