# SoccerNote - 축구팀 경기 기록 MVP 기획안

## 1. 프로젝트 개요

### 주제
축구팀 감독이 경기 후 선수별 평가를 기록하고, 쿼터별 포메이션과 성과를 관리하는 앱

### 타겟 사용자
- **주요 사용자**: 아마추어/동호회 축구팀 감독 또는 코치
- **사용 시나리오**: 경기 종료 후 바로 선수별 점수와 기록을 남기고, 시즌 MVP를 선정

### 핵심 가치 제안
1. 쿼터별 선수 평가를 직관적인 UI로 빠르게 기록
2. 드래그 앤 드롭으로 실제 포메이션을 시각적으로 구성
3. 자동 MVP 계산으로 객관적인 선수 평가

### 차별화 포인트
기존 축구 기록 앱은 단순 점수 기록에 그치지만, SoccerNote는 쿼터별 포메이션 시각화와 드래그 앤 드롭 선수 배치, 자동 MVP 산정 기능을 제공

---

## 2. 핵심 기능 (MVP 범위)

### 필수 기능

1. **경기 생성 및 관리**
   - 설명: 새 경기 생성 (상대팀, 날짜, 장소), 4쿼터 구성
   - 사용자 스토리: "감독으로서, 경기를 시작하기 전에 기본 정보를 입력하고 싶다"
   - 우선순위: High

2. **포메이션 시각화 및 선수 배치**
   - 설명: 축구 필드 UI에서 11명의 선수를 드래그 앤 드롭으로 포지션에 배치
   - 포지션: 골키퍼(GK), 수비(DF), 미드필더(MF), 공격수(FW)
   - 사용자 스토리: "감독으로서, 각 쿼터별 포메이션을 시각적으로 구성하기 위해 선수를 드래그해서 배치하고 싶다"
   - 우선순위: High

3. **쿼터별 선수 평가**
   - 설명: 각 쿼터(25분 x 4)마다 11명 선수에 대해 0.0~10.0 점수 입력
   - 사용자 스토리: "감독으로서, 각 쿼터마다 선수들의 활약도를 점수로 남기고 싶다"
   - 우선순위: High

4. **선수 통계 입력**
   - 설명: 골, 어시스트, 클린시트, 기여도 수치 입력
   - 사용자 스토리: "감독으로서, 선수의 구체적인 성과 지표를 기록하고 싶다"
   - 우선순위: High

5. **MVP 자동 산정**
   - 설명: 모든 쿼터 평균 점수가 가장 높은 선수를 MVP로 자동 선정
   - 사용자 스토리: "감독으로서, 객관적인 데이터로 MVP를 선정하고 싶다"
   - 우선순위: High

### 향후 고려 기능 (MVP 이후)
- 시즌 전체 통계 및 리포트
- 선수 프로필 사진 업로드
- 팀 간 경기 기록 비교
- 영상 클립 연동

## 4. 데이터 구조 개요

### 주요 엔티티

1. **Team (팀)**
   - 주요 속성: id, name, user_id (감독), created_at
   - 관계: 1:N Players, 1:N Matches

2. **Player (선수)**
   - 주요 속성: id, team_id, name, default_position
   - 관계: N:1 Team, 1:N QuarterRecords

3. **Match (경기)**
   - 주요 속성: id, team_id, opponent, match_date, location
   - 관계: N:1 Team, 1:N Quarters

4. **Quarter (쿼터)**
   - 주요 속성: id, match_id, quarter_number (1-4), formation_data (JSON)
   - 관계: N:1 Match, 1:N QuarterRecords

5. **QuarterRecord (쿼터별 선수 기록)**
   - 주요 속성: id, quarter_id, player_id, position_x, position_y, position_type, rating (0.0-10.0), goals, assists, clean_sheet, contribution
   - 관계: N:1 Quarter, N:1 Player

### 데이터 플로우
```
감독 로그인 → 팀 선택 → 경기 생성 → 쿼터별 선수 배치 → 점수/통계 입력 → MVP 확인
```

## 5. 사용자 플로우

### 주요 사용자 여정

1. **첫 방문 사용자**
   ```
   랜딩 페이지 → 회원가입 → 팀 생성 → 선수 등록 → 첫 경기 생성
   ```

2. **재방문 사용자 (경기 후 기록)**
   ```
   로그인 → 대시보드 → 새 경기 → 쿼터별 포메이션/점수 입력 → 저장 → MVP 확인
   ```

### 핵심 액션 플로우
- **경기 기록**: 새 경기 생성 → 1쿼터 선수 배치 → 점수 입력 → 2~4쿼터 반복 → 저장
- **MVP 확인**: 경기 상세 → 전체 쿼터 평균 계산 → MVP 하이라이트 표시

## 6. 화면 구성

### 필수 페이지
1. `/` - 랜딩 페이지
2. `/login` - 로그인
3. `/signup` - 회원가입
4. `/dashboard` - 팀 대시보드 (경기 목록)
5. `/team/players` - 선수 관리
6. `/match/new` - 새 경기 생성
7. `/match/[id]` - 경기 상세 (쿼터별 기록)
8. `/match/[id]/quarter/[q]` - 쿼터 편집 (포메이션 + 점수)

### 화면별 역할
| 페이지 | 역할 | 인증 필요 |
|--------|------|-----------|
| 랜딩 | 앱 소개, 로그인 CTA | X |
| 대시보드 | 경기 목록, 팀 요약 | O |
| 선수 관리 | 선수 CRUD | O |
| 경기 상세 | 쿼터별 점수, MVP 표시 | O |
| 쿼터 편집 | 드래그 앤 드롭 포메이션, 점수 입력 | O |
